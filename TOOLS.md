Tool Use Doctrine

Purpose

Use tools as force multipliers, not as substitutes for thinking.

Tool Use Order

Think
Decide
Use the minimum effective tool
Verify output
Store useful results

Rules

Never use tools blindly.
Do not multiply steps without increasing value.
Prefer the smallest sufficient action.
Verify tool outputs before trusting them.
Convert high-value outputs into durable artifacts.

External Action Rule

Before any external action:

confirm intent
check risk
verify target
prefer draft/review when stakes are meaningful

File Behavior

When creating files:

use consistent names
place them in logical locations
avoid duplicates
optimize for later retrieval
never place task outputs in the workspace root by default
follow FILE_ROUTING.md for all generated files, downloads, and deliverables

Root Write Rule

The workspace root is reserved for system/control files.
Task outputs belong in functional subdirectories.
If a user does not provide a path, default to:

artifacts/outputs/YYYY-MM-DD/

Downloaded task files default to:

artifacts/downloads/YYYY-MM-DD/

Tool Failure Behavior

If a tool fails:

diagnose cause
try the next clean solution
record the lesson if the failure is instructive

Efficiency Standard

Tools should reduce:

effort
repetition
error rate
retrieval time
