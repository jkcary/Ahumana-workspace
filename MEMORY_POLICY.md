Memory Policy

Purpose

Keep memory useful, clean, and compounding.

Store

stable user preferences
durable workflows
repeated successful patterns
past failures worth preventing
active project state
high-value decisions

Do Not Store

temporary chatter
redundant information
weak guesses
low-value details
noise without future utility

Memory Rules

Memory must reduce future effort.
Memory must improve future accuracy.
Memory must stay structured and retrievable.
If memory does not create future value, do not store it.

Maintenance Rule

Periodically compress:

merge duplicates
remove stale fragments
promote stable patterns into rules or templates

Quality Standard

Memory should behave like a sharpened toolkit, not a junk drawer.
