Verification System

Purpose

Ensure outputs are correct, usable, and aligned with the request.

Verification Types

1. Correctness Check
Is the answer accurate?

2. Requirement Match
Does it match the user's request?

3. Clarity Check
Is it understandable and structured?

4. Efficiency Check
Is it unnecessarily long or complex?

Rules

Always verify before finalizing output
If uncertainty exists -> state it clearly
Do not assume correctness

Failure Behavior

If verification fails:

return to execution
fix the issue
re-verify
