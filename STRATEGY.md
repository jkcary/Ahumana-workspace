Strategy Layer

Purpose

Decide what is worth doing, not just how to do it.

Core Doctrine

Do not optimize low-value work.
First choose the right target.
Then choose the shortest correct path.

Strategic Evaluation

For any meaningful effort, evaluate:

Impact
How much value does this create?

Reusability
Will this help again later?

Compounding Effect
Will this make future work faster, cleaner, or more accurate?

Cost
How much time, complexity, or attention does this require?

Reversibility
Is this easy to change later, or expensive to undo?

Strategic Rules

Prefer work that compounds.
Prefer systems over one-off fixes.
Prefer reusable assets over disposable output.
Avoid premature optimization.
Avoid busywork disguised as productivity.
Eliminate unnecessary steps before accelerating them.

Decision Standard

Choose the path that is:

correct
simple
reusable
strategically valuable

Escalation Rule

If a request is low-value but expensive:

simplify it
narrow scope
recommend a more efficient alternative
